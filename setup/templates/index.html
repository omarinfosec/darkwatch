Telegram credentials

From my.telegram.org/apps. Stored in /var/lib/darkwebapp/env as TELEGRAM_API_ID + TELEGRAM_API_HASH. {% if state.telegram_api_hash_set %}A hash is already stored — you must re-paste it here to change it (we never show the saved value).{% endif %} Before saving, credentials are checked against Telegram {% if state.tunnel2_conf_present %}(via Tunnel 2){% else %}(direct if Tunnel 2 is not configured yet){% endif %}. Saving recreates the darkwatch container so it picks up the new environment variables (a plain restart is not enough).

Tunnel 1 — Tor research path

WireGuard config that carries the dark-web crawl traffic. Egress through this tunnel reaches Tor entry guards. Paste the contents of your provider's wg0.conf below. It must contain [Interface] with a PrivateKey and [Peer] with a PublicKey + Endpoint. Saving restarts tunnel1 + tor + darkwatch.

Tunnel 2 — Telegram research path

WireGuard config that carries Telegram MTProto traffic. Use a different region / provider than Tunnel 1 — sharing an exit IP across the two paths defeats the egress isolation. Saving restarts tunnel2 + tg-socks + darkwatch.

Bring this UI down when you're done: docker compose --profile setup stop setup. Token persists in /var/lib/darkwebapp/env; rotate via openssl rand -hex 32.
